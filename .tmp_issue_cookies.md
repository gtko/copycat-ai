## Description
Les cookies de session n'utilisent pas les attributs de sécurité recommandés pour la production.

### Localisation
- `src/routes/auth.ts` (login, verify, logout)
- `src/index.ts` (checkAuth)

### Code actuel
```typescript
'Set-Cookie': `session=${sessionId}; HttpOnly; Path=/; Max-Age=604800; SameSite=Lax`
```

### Problèmes
1. **Pas d'attribut `Secure`**: Cookie envoyé sur HTTP non sécurisé
2. **Pas de prefix `__Host-`**: Vulnérable au cookie tossing
3. **SameSite=Lax**: Potentiellement insuffisant pour une API

### Solution recommandée
```typescript
// En production
const cookie = `__Host-session=${sessionId}; HttpOnly; Secure; Path=/; Max-Age=604800; SameSite=Strict`;

// En dev (localhost)
const cookie = `session=${sessionId}; HttpOnly; Path=/; Max-Age=604800; SameSite=Lax`;
```

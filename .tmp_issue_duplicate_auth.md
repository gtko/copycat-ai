## Description
La logique de parsing des cookies est dupliquée et fragile (utilisation de regex simple).

### Localisation
- `src/index.ts` ligne ~52: `checkAuth()`
- `src/routes/auth.ts` ligne ~76: `/me`
- `src/routes/auth.ts` ligne ~95: `/logout`
- `src/routes/stripe.ts` ligne ~74: `/portal`
- `src/routes/api.ts` ligne ~12: middleware auth

### Code dupliqué
```typescript
const sessionId = c.req.header('cookie')?.match(/session=([^;]+)/)?.[1];
```

### Problèmes
1. **Duplication**: Même code copié 5 fois
2. **Fragilité**: Ne gère pas les espaces (`session = value`)
3. **Pas d'abstraction**: Difficile à maintenir

### Solution recommandée
Créer un helper réutilisable:
```typescript
// src/utils/auth.ts
export function getSessionId(c: Context): string | undefined {
  const cookie = c.req.header('cookie');
  if (!cookie) return undefined;
  
  const match = cookie.match(/session=([^;]+)/);
  return match?.[1];
}

export async function getUser(c: Context): Promise<User | null> {
  const sessionId = getSessionId(c);
  if (!sessionId) return null;
  
  return c.env.DB.prepare(
    'SELECT u.id, u.email...'
  ).bind(sessionId).first();
}
```
